#  Backup-ийн төрлүүд

Backup гэдэг нь өгөгдлийг алдагдах, гэмтэхээс хамгаалах зорилгоор хуулбарлан хадгалах үйл явц юм.

## 1. Full Backup (Бүрэн нөөцлөлт)

**Тайлбар:**  
Бүх өгөгдлийг нэг дор бүрэн хуулбарлаж хадгална.

**Давуу тал:**
- Сэргээхэд хамгийн хурдан
- Бүх мэдээлэл нэг дор байна

**Сул тал:**
- Их хэмжээний storage ашиглана
- Backup хийхэд удаан

---

## 2.  Incremental Backup (Нэмэгдэл нөөцлөлт)

**Тайлбар:**  
Сүүлийн backup-аас хойш өөрчлөгдсөн өгөгдлийг л хадгална.

**Давуу тал:**
- Storage бага ашиглана
- Backup хурдан

**Сул тал:**
- Сэргээхэд олон файл шаардлагатай
- Сэргээх процесс удаан

---
## 3. Differential Backup (Ялгаварт нөөцлөлт)

**Тайлбар:**  
Сүүлийн full backup-аас хойш өөрчлөгдсөн бүх өгөгдлийг хадгална.

**Давуу тал:**
- Incremental-аас сэргээхэд хурдан
- Дунд зэргийн storage хэрэглээ

**Сул тал:**
- Backup хэмжээ аажмаар томорно

---

## 4. Mirror Backup (Толь нөөцлөлт)

**Тайлбар:**  
Өгөгдлийг яг адилхан байдлаар realtime эсвэл шууд хуулбарлана.

**Давуу тал:**
- Шууд ашиглах боломжтой
- Сэргээх шаардлагагүй шахуу

**Сул тал:**
- Алдаа (жишээ нь файл устгах) шууд backup-д нөлөөлнө

---
## 5.  Cloud Backup (Үүлэн нөөцлөлт)

**Тайлбар:**  
Өгөгдлийг интернетээр дамжуулан cloud серверт хадгална.

**Давуу тал:**
- Хаанаас ч хандах боломжтой
- Hardware эвдрэлээс хамгаална

**Сул тал:**
- Интернет шаардлагатай
- Аюулгүй байдал анхаарах хэрэгтэй

---

## 6. Hybrid Backup (Хосолсон нөөцлөлт)

**Тайлбар:**  
Local + Cloud backup-ийг хослуулсан хэлбэр.

**Давуу тал:**
- Давхар хамгаалалт
- Хурдан + найдвартай

**Сул тал:**
- Илүү зардалтай
- Удирдах төвөгтэй

---
##  Дүгнэлт

Backup-ийн төрлүүд нь өгөгдлийг хамгаалах өөр өөр арга барилтай бөгөөд тухайн хэрэгцээнээс хамаарч сонгох нь чухал юм.  

- **Full backup** нь хамгийн найдвартай боловч их нөөц шаарддаг  
- **Incremental backup** нь зай хэмнэдэг ч сэргээхэд төвөгтэй  
- **Differential backup** нь дунд зэргийн шийдэл болдог  
- **Mirror backup** нь шууд хуулбар боловч алдаа давхар дамжуулдаг  
- **Cloud backup** нь алсын хадгалалт, уян хатан байдлыг өгдөг  
- **Hybrid backup** нь хамгийн найдвартай боловч зардал ихтэй  

Иймээс байгууллага болон хэрэглэгчид өөрсдийн **өгөгдлийн хэмжээ, аюулгүй байдал, зардал, сэргээх хурд**-д тохируулан backup-ийн төрлийг зөв сонгох хэрэгтэй.

##  Backup гэж юу вэ? (Дэлгэрэнгүй тайлбар)

**Backup (нөөцлөлт)** гэдэг нь аливаа өгөгдөл (файл, зураг, видео, database, системийн мэдээлэл гэх мэт)-ийг  
алдах, гэмтэх, устахаас сэргийлэх зорилгоор **өөр газар, өөр хэлбэрээр хуулбарлан хадгалах үйл явц** юм.

---

##  Backup хийх зорилго

Backup нь дараах эрсдлээс хамгаална:

- **Hardware эвдрэл** (хатуу диск гэмтэх)
- **Хүний алдаа** (санамсаргүй устгах)
- **Вирус, ransomware халдлага**
- **Байгалийн гамшиг** (гал, ус, газар хөдлөлт)
- **Цахилгааны доголдол**

---

##  Backup хэрхэн ажилладаг вэ?

Backup хийх үед:
1. Анхны өгөгдлийг (source data) сонгоно  
2. Түүний хуулбарыг үүсгэнэ  
3. Тэр хуулбарыг өөр хадгалах орчинд байрлуулна  
   - External hard disk  
   - Cloud storage  
   - Server  

Ингэснээр эх өгөгдөл устсан ч backup-аас сэргээх боломжтой болдог.

---

##  Backup-д хамрагдах зүйлс

-  Баримт бичиг (Word, PDF, Excel)
-  Зураг, видео
-  Database
-  Системийн тохиргоо (OS backup)
-  Вэбсайт файлууд

---

##  Backup vs Restore

- **Backup** → Өгөгдлийг хуулж хадгалах  
- **Restore** → Backup-аас өгөгдлийг буцааж сэргээх  

---

## Backup хийх зарчим (3-2-1 дүрэм)

Найдвартай backup хийхэд дараах дүрмийг ашигладаг:

- **3** → 3 хуулбар өгөгдөлтэй байх  
- **2** → 2 өөр төрлийн хадгалалт (жишээ: HDD + Cloud)  
- **1** → 1 нь заавал өөр газар (offsite) хадгалагдсан байх  

---

## Жишээ

Та laptop дээрээ дипломын ажлаа хадгалж байна гэж бодъё:

- Original файл → Laptop дээр  
- Backup 1 → External hard disk  
- Backup 2 → Google Drive (Cloud)  

Хэрэв laptop эвдэрвэл → backup-аас файлаа сэргээнэ

---

##  Яагаад backup чухал вэ?

Backup байхгүй бол:
- Өгөгдөл бүрэн алдагдана  
- Сэргээх боломжгүй болно  
- Санхүүгийн болон цаг хугацааны алдагдал гарна  

---

## Дүгнэлт

Backup нь өгөгдлийг хамгаалах хамгийн чухал арга бөгөөд  
ямар ч систем, хэрэглэгчийн хувьд **заавал хэрэгжүүлэх ёстой хамгаалалтын механизм** юм.
